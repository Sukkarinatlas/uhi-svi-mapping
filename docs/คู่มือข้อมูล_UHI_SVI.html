หัวข้อที่ 9 — Urban Heat Island and Social Vulnerability Mapping Application
แอปพลิเคชันวิเคราะห์เกาะความร้อนเมือง ประมวลผลอุณหภูมิพื้นผิว (LST) ซ้อนทับข้อมูลประชากร เพื่อระบุพื้นที่วิกฤตทางความร้อนที่มีความเปราะบางทางสังคมสูง — ขอบเขต: ทั่วประเทศไทย

กลุ่มที่ 3: Environment & Social Security | จัดทำโดย Claude (Cowork) · 22 กรกฎาคม 2569

1. สรุปแนวคิดของแอป

แอปนี้ตอบโจทย์ด้วยการซ้อน 3 ชั้นข้อมูลหลักเข้าด้วยกัน:

  • ชั้นความร้อน (Heat): อุณหภูมิพื้นผิว (Land Surface Temperature) จากดาวเทียม → คำนวณดัชนีเกาะความร้อนเมือง (UHI Intensity = LST พื้นที่เมือง − LST พื้นที่ชนบทอ้างอิง)
  • ชั้นความเปราะบางทางสังคม (Social Vulnerability): ความหนาแน่นประชากร + สัดส่วนกลุ่มเปราะบาง (ผู้สูงอายุ เด็กเล็ก) + ฐานะเศรษฐกิจ + การเข้าถึงบริการสุขภาพ
  • ผลลัพธ์ (Hotspot วิกฤต): พื้นที่ที่ ความร้อนสูง × ความเปราะบางสูง = พื้นที่ต้องแทรกแซงก่อน (priority zone)

2. หมวดข้อมูลและวิธีเข้าถึง

รวบรวมได้ 40 แหล่ง แบ่ง 5 หมวด รองรับทั้ง GEE WMS/WFS REST API Static Tile (รายละเอียดครบในไฟล์ catalog_UHI_SVI.csv):

A. อุณหภูมิพื้นผิว / ความร้อน (LST) — อินพุตหลัก

เลือกตามความละเอียดที่ต้องการ: Landsat 8/9 (30 ม.) ดีที่สุดสำหรับระดับย่านเมือง; MODIS (1 กม.) ให้ความถี่รายวันเหมาะทำ climatology; ECOSTRESS (~70 ม.) ให้หลายช่วงเวลาของวัน; ERA5-Land เสริมอุณหภูมิอากาศเพื่อคำนวณ Heat Index

B. สิ่งแวดล้อม / การใช้ที่ดิน — ปัจจัยขับ UHI

Dynamic World และ ESA WorldCover (10 ม.) จำแนกพื้นที่สิ่งปลูกสร้าง vs พื้นที่สีเขียว; Sentinel-2 คำนวณ NDVI/NDBI; GHSL Built-up และ Open Buildings ให้ความหนาแน่นสิ่งปลูกสร้าง

C. ประชากร / ความเปราะบางทางสังคม

WorldPop (100 ม. รวมโครงสร้างอายุ-เพศ) เป็นแกนหลัก; เสริมด้วย GHS-POP, GPW v4.11; Relative Wealth Index และ HRSL ของ Meta เป็นตัวแทนฐานะเศรษฐกิจ-สังคม

D. หน่วยงานไทย / ข้อมูลราชการ

DOPA (ประชากรทะเบียนราษฎรระดับตำบล แยกอายุ) และ NSO (สำมะโน 2563) เป็นข้อมูลความเปราะบางที่แม่นระดับพื้นที่; GISTDA sphere ให้ Web Service ภูมิสารสนเทศไทย + THEOS-2; HDX/GADM ให้ขอบเขตตำบล; TMD/Air4Thai/HDC ให้ข้อมูลเรียลไทม์เชิงสุขภาพ

E. Basemap / โครงสร้าง Web Service

Longdo Map (ภาษาไทย), OSM, Esri World Imagery เป็น basemap; ผลวิเคราะห์จาก GEE เผยแพร่เป็น XYZ tile ผ่าน getMapId ลงเว็บได้โดยตรง

หมายเหตุสำคัญ: MODIS ใกล้ปลดระวาง — วางแผนใช้ VIIRS (VNP21) ระยะยาว · ข้อมูลราชการไทย (DOPA/NSO/GISTDA) มีความละเอียดเชิงพื้นที่ระดับตำบล จึงควรใช้เป็นหน่วยรายงานผล (aggregate) ส่วนดาวเทียมใช้เป็นชั้นวิเคราะห์เชิงกริด

3. แนวทางคำนวณ Social Vulnerability Index (SVI) สำหรับไทย

ไทยยังไม่มี SVI มาตรฐานระดับพื้นที่ จึงเสนอสร้างดัชนีรวม (composite) จากตัวชี้วัดต่อไปนี้ แล้ว normalize (0–1) และถ่วงน้ำหนัก:

  • % ผู้สูงอายุ (65+) และ % เด็กเล็ก (<5) — จาก WorldPop age/sex หรือ DOPA
  • ความหนาแน่นประชากร — WorldPop / GHS-POP
  • ฐานะเศรษฐกิจ (RWI) หรือรายได้ครัวเรือน (NSO)
  • ระยะทางถึงสถานพยาบาลที่ใกล้ที่สุด — OSM/HDC
  • สัดส่วนพื้นที่สีเขียว (NDVI ต่ำ = เปราะบางกว่า)

ดัชนีวิกฤต (Priority) = UHI Intensity (normalize) × SVI → จัดชั้น High / Medium / Low ตามภาพในสไลด์

4. ตัวอย่างโค้ด Google Earth Engine (Landsat LST + WorldPop)

// LST จาก Landsat 8/9 (Collection 2 L2) เฉลี่ยฤดูร้อน ทั่วไทย
var thai = ee.FeatureCollection('FAO/GAUL/2015/level0')
             .filter(ee.Filter.eq('ADM0_NAME','Thailand'));

function toLST(img){
  return img.select('ST_B10').multiply(0.00341802).add(149.0)
            .subtract(273.15).rename('LST_C')     // → องศาเซลเซียส
            .copyProperties(img, ['system:time_start']);
}
var l89 = ee.ImageCollection('LANDSAT/LC08/C02/T1_L2')
  .merge(ee.ImageCollection('LANDSAT/LC09/C02/T1_L2'))
  .filterBounds(thai)
  .filterDate('2024-03-01','2024-05-31')     // ฤดูร้อน
  .filter(ee.Filter.lt('CLOUD_COVER',20))
  .map(toLST);
var lst = l89.select('LST_C').mean().clip(thai);

// ประชากรเปราะบาง: WorldPop ผู้สูงอายุ (ตัวอย่างรวมกลุ่มอายุ 60+)
var pop = ee.ImageCollection('WorldPop/GP/100m/pop')
  .filter(ee.Filter.eq('country','THA'))
  .filter(ee.Filter.eq('year',2020)).first();

// พื้นที่วิกฤต = LST สูง และ ประชากรหนาแน่น
var hot = lst.gt(38);                 // เกณฑ์ความร้อน (ปรับได้)
var dense = pop.gt(50);
var critical = hot.and(dense);

Map.addLayer(lst,{min:25,max:45,palette:['blue','yellow','red']},'LST (°C)');
Map.addLayer(critical.selfMask(),{palette:['#8B0000']},'พื้นที่วิกฤต');

// เผยแพร่เป็น XYZ tile ลงเว็บ (Leaflet/MapLibre)
var mapid = lst.getMapId({min:25,max:45,palette:['blue','yellow','red']});
print(mapid.urlFormat);   // นำ URL นี้ไปใส่เป็น tile layer

5. สถาปัตยกรรม Web Map App ที่แนะนำ

  • ประมวลผล: Google Earth Engine (Python API + ee.data.getMapId) เป็น backend คำนวณ LST/UHI/SVI แบบ on-the-fly
  • Front-end แผนที่: Leaflet หรือ MapLibre GL + basemap Longdo/OSM
  • ชั้นวิเคราะห์: ต่อ XYZ tile จาก GEE (dynamic) + WMS ของ GISTDA
  • ชั้นขอบเขต/สถิติ: GeoJSON ตำบล (HDX/GADM) join กับ DOPA/NSO ทำ choropleth และ dashboard รายพื้นที่
  • ข้อมูลเรียลไทม์: ดึง TMD (อุณหภูมิ) และ Air4Thai (คุณภาพอากาศ) ผ่าน REST API
